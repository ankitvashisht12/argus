### Sample model output (summary)

> `assertArray` accepts an optional per-element assertion function. It used to
> invoke it via `value.forEach(assertion)`, and since `forEach` hands its callback
> (element, index, array), the numeric index landed in the assertion's `message`
> parameter — so any element failure threw with a message like "1" instead of the
> caller's message or the library's default. This PR replaces the `forEach` with an
> explicit `for…of` loop calling `assertion(element, message)`, widens the
> `assertion` parameter type, and silences the resulting compiler error with a
> `@ts-expect-error`. One AVA assertion is added checking the caller-supplied
> message is thrown.

(Accurate against the actual PR diff — the anchored notes landed on both real hunks.)
