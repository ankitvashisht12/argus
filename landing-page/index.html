how it works · the pipeline

Argus turns that wall of diff into a guided review. Here's the pipeline, pass by pass — the passes v1 runs today, and the two the upcoming release adds.

  1. 00

    point it at a branch

    One binary, your machine, your subscription.

    Run argus in any git repo and confirm the base and head. It reads the diff locally through git — no GitHub dependency, no upload, and none of GitHub's 20k-line API cap — shelling out to your already-authenticated Claude CLI for the thinking.

  2. 01

    infer the intent

    Reviews against intent, not guesses.

    Argus infers what the change is actually for — from the diff itself and the repo files it reads for context — and grounds every explanation in that intent, so a deliberate choice reads as intentional instead of being mistaken for a slip. It reads past the diff for what the diff alone can't show: where moved code went, and what later code leans on it.

  3. 02

    a reading order, not an alphabet

    Read it in the order it was built.

    Before you read a line, Argus writes a whole-change brief and reorders the file tree into a dependency-sensible path — entry points and core logic first, generated files and lockfiles last.

  4. 03

    Upcoming release review every file, then refute it

    Findings that survive a pass built to disprove them.

    The upcoming release runs correctness, security, and test lenses over every file, then an adversarial self-check pass that tries to refute each finding and drops the ones that don't survive — so what stays flagged is worth your attention, and the rest is safe to skim.

  5. 04

    Upcoming release run your actual tests

    The repo's own tests, turned into findings.

    The upcoming release auto-detects and runs the repo's own test, lint, and build commands, turns real failures into findings, and flags a change that ships no tests as a risk of its own.

  6. 05

    the review surface

    Explanations inline, Claude a keystroke away.

    Why-it-changed explanations render directly above each hunk (marked ▌ so they're never mistaken for a +/− line), the file tree keeps the dependency-sensible reading order with generated files and lockfiles pushed last, and a streaming Claude chat is one keystroke away for any line you don't follow — in the terminal today, in a local web UI next.

    needs you

    The explanation puts the one thing worth confirming right in front of you: check that the widened key class won't mask a legitimate value that just happens to contain “key”. Everything else in the file is safe to skim.

Once you know why a change was made, most of the diff is safe to skim.

the trade-off

It runs on your machine. Not in someone else's cloud.

Argus is a single Go binary that shells out to git and your authenticated Claude CLI — the opposite trade from cloud review bots like CodeRabbit and Greptile, which read your PR on someone else's servers. It infers intent, explains why each change was made, and hands a human a guided reading order instead of a wall of diff.

  • no upload

    Your code never leaves the laptop. No server, no third-party service.

  • your subscription

    Drives the Claude CLI you already pay for. No per-seat SaaS.

  • no api cap

    Reads the diff locally through git — so GitHub's 20k-line API diff cap never applies.

  • stop & resume

    Quit any time; your progress and cached explanations persist, so you pick up where you left off.
